Order vs inventory reconciliation

Compare email knife orders to inventory_items_v2. Traditions, Ultra, and Damascus catalog models match on model only (handle/blade in the DB ignored for gaps). Full mode uses blade detail only for Blood Brothers and tactical models; other knives use model + handle + blade *. Tactical two-tones (e.g. COY/OD): first = blade, second = handle. If an inventory row has MKC order # (field or in notes) that matches an email line for the same model, that quantity is netted out of bucket gaps. If an inventory row has an acquired_date (order/purchase date) but its order #(s) are not present in the email CSV, that inventory quantity is excluded from bucket gap math (treat as already matched). Trailing VIP gifts are excluded from inventory counts. Mark rows that are not real discrepancies and link order lines to inventory pieces.
