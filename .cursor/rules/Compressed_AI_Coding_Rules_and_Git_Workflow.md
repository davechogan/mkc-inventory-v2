# Compressed AI Coding Rules and Git Workflow

Cursor loads **project rules** from `.mdc` files in this folder (YAML frontmatter + body). The content of this document is split into:

| Rule file | Topic |
|-----------|--------|
| `engineering-standards.mdc` | Canonical layers, architecture, anti-patterns |
| `testing-and-nlp-quality.mdc` | Tests, planner/NLP invariants, bugfix matrix |
| `ai-change-checklist.mdc` | Questions to answer before finalizing a change |
| `git-workflow.mdc` | Branches, commits, isolation, merges |

**Edit the `.mdc` files** to change what the agent follows. This `.md` file is only a human-readable index.

See also: `python-env-and-ci.mdc` (venv and local CI before commits).
